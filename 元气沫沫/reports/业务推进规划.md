## 二、Q3重点任务（7件）

### 🔴 任务一：智能订货系统（7月）

**目标**：用数据替代经验，降低报损率至3%以下

**背景**：
- 当前订货流程完全依赖店长人工判断
- 朝阳站三次长假报损失控（五一/六一/端午）
- 6月报损¥19,033虽大幅改善但仍占收入3.5%

**方案**：
```
输入：历史销量(18月) + 天气 + 节假日 + 实时库存
  ↓
算法：品类级需求预测（移动平均+节假日系数+天气因子）
  ↓
输出：每门店/每品类每日建议订货量
```

**技术架构**：
- 数据层：从经营数据总览Excel + 库存监测表 定时抽取
- 算法层：Python脚本（codex exec），逐步从简单移动平均升级到时序预测
- 输出层：每日8:00前生成订货建议表，推送至工作群

**预期效果**：
- 报损率从3.5%降至2.5%以内（月省¥5,000+）
- 缺货率降低（释放被库存不足抑制的销售）

**里程碑**：
- Week 1：数据管道搭建，历史数据清洗
- Week 2：基准模型上线（移动平均+工作日/周末因子）
- Week 3-4：加入天气/节假日因子，朝阳站专项优化

### 🔴 任务二：客单价回升方案（7-8月）

**目标**：客单价从¥12.11回升至¥13+

**策略组合**：

#### A. 套餐设计（短期见效）
```
三明治(¥11-13) + 饮品(¥5-6) = 套餐价¥15（原价¥16-19）
```
- 选品：TOP3三明治（香煎培根/奥尔良/蒸蛋火腿）+ TOP5饮品
- 试点：朝阳站 + 平安里（高客流门店）
- 预期：套餐占比达20%时，整体客单价提升¥0.5-1

#### B. 价格带升级（中期）
- 引入¥15-18价格带新品（顾问提供：日式照烧鸡肉包、油浸小番茄恰巴塔）
- 逐步淘汰¥3-5的低毛利饮品（Costa拿铁、三得利系列）

#### C. 会员价策略
- 调研美团后台会员管理功能
- 设计"早餐卡"：¥99/月，每日任选三明治+饮品一份（锁定高频用户）

### 🟡 任务五：朝阳站节假日备货模型（7月完成）

**背景**：朝阳站是高铁站门店，节假日客流波动极大（6/18高峰¥16,614 → 6/20骤降至¥3,794）

**方案**：
1. 建立历史数据参考库：纳入五一(5/1-5)、六一(6/1)、端午(6/19)、国庆、春节、清明
2. 备货规则：
   - 节前2天：备货=平日1.5倍（不超2倍）
   - 节日当天：保持平日1.2倍
   - 节后第1天：立即降至平日0.7倍（防止惯性备货）
3. 更新机制：每次长假后复盘报损数据，更新参考库

**预期效果**：长假报损从¥2,000+/次降至¥500以内

### 🟢 任务六：盘点异常监测（8月）

**目标**：降低盘点差异率，减少货损

**方案**：
- 在现有小时监测表基础上，增加"异常标记"功能
- 规则引擎：
  - 单品当日销量 > 历史均值3σ → 异常
  - 库存结余率 < 10% 或 > 80% → 异常
  - 同品类多门店同时缺货 → 配送异常
- 输出：每日异常清单，推送区经复核
